aexGrabber

This is a module that extracts independent event-driven response to changes in the luminance sensed

Event driven asynchronous sensors transmit the local pixel-level changes caused by movement in a scene at the time they occur. The result is a stream of events at microsecond time resolution with very low redundancy that drastically reduces power, data storage and computational requirements.

Parameters

Command-line Parameters

The following key-value pairs can be specified as command-line parameters by prefixing – to the key (e.g. –from file.ini. The value part can be changed to suit your needs; the default values are shown below.

  • from aexGrabber.ini
    specifies the configuration file
  • context aexGrabber/conf
    specifies the sub-path from $ICUB_ROOT/icub/app to the configuration file
  • name aexGrabber
    specifies the name of the module (used to form the stem of module port names)
  • robot icub
    specifies the name of the robot (used to form the root of robot port names)
